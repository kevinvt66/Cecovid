Todo sobre las variables

Formas de declarar variables

  • Usando var delante
  • Usando let delante
  • Usando const delante
  • Usando nada

Por buenas prácticas, usaremos siempre una de las 3 primeras.

var

var es la forma clásica de declarar una variable desde los inicios de JavaScript.

let y const fueron añadidas al lenguaje en 2015.

var se usa principalmente como método de compatibilidad con navegadores antiguos.

const y let

Por regla general, las variables se declaran con const.

Sin embargo, si el valor de las variables puede cambiar, usaremos let.

El operador de asignación =

En JavaScript, el signo de igual = no equivale a "resultado de" como en álgebra. Si no que es un operador de asignación.

x = x + 5

Esta expresión, matemáticamente, no tiene sentido. Pues nunca va a haber un número equivalga a una suma de sí mismo con 5.

Sin embargo, en código, no sólo tiene mucho sentido si no que es una práctica muy común. Esta expresión indica que al actual valor de x le estamos asignando el valor de la suma de x + 5.

Básicamente, hemos aumentado el valor de x en 5.

Nota: En JavaScript el operador "igual a" se escribe ==.

Tipos de datos

Las variables pueden contener distintos tipos de datos. Pueden ser número como 100 o texto como "Juan Eduardo".

En programación, a los valores de texto se les llama cadenas de caracteres o strings.

Los strings se escriben dentro de comillas dobles o simples.

Si escribes un número dentro de unas comillas, será tratado como un string de texto. Ref al js línea 15

Value = undefined

A menudo en los programas informáticos, las variables se declaran sin un valor. El valor puede que sea algo que deba ser calculado o que sea provisto por el usuario, como un input.

Las variables declaradas sin valor tendrán como valor undefined. Ref al js línea 25.

Redeclarar variables

Si se redeclara una variable usando var, la variable mantendrá su valor.

Sin embargo, al hacer esto con una variable declarada con let o cosnt el código lanzará un error.
